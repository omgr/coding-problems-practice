## How It Works

### Timing
- **Every 3 problems**: After completing 3 practice problems, you'll take a timed challenge
- **Random language**: I'll randomly select one of your 5 languages
- **Time limits**: 30-60 minutes depending on difficulty
- **Real interview conditions**: Simulates actual technical interviews

### Schedule

## What Gets Stored Here

When you complete a timed challenge, you'll save:
- `challenge-1-[date].py` (or .js, .cs, .go, .ts)
- `challenge-1-notes.md` - Your approach, struggles, learnings
- `challenge-1-results.txt` - Time taken, test results

1. **Set a timer** - Stick to the time limit
2. **Think out loud** - Practice explaining as you code
3. **Write tests first** - Plan before coding
4. **Don't give up** - If stuck, explain what you'd do with more time
